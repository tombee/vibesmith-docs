## Why an address layer

A common pain shape: rename `assets/images/icons/cursor.png` →
`assets/images/icons/pointer.png` and every code reference snaps.
Unity solved this with GUIDs in `.meta` files; Unreal with binary
`.uasset` containers. Vibesmith uses a **text-diffable address
layer**: every asset has a stable `kind:ref` string carried in
its sidecar; renames change the path but not the address.

The runtime, the editor, MCP tools, and AI coding agents all
reason about assets by *address*, never by path.

## Address shape

`kind:ref`. The default address is derived from the asset's
source path:

| Source path                         | Default address          |
| ----------------------------------- | ------------------------ |
| `assets/images/icons/cursor.png`    | `image:icons/cursor`     |
| `assets/audio/music/menu-loop.opus` | `audio:music/menu-loop`  |
| `assets/fonts/body.woff2`           | `font:body`              |
| `assets/meshes/characters/hero.glb` | `mesh:characters/hero`   |

Format rules:

- **`kind`** — one of `image`, `audio`, `font`, `mesh`. Closed
  enum at v1; new kinds are additive (`video`, `image-atlas`,
  `text-data`, `binary-data` reserved).
- **`ref`** — namespaced under `kind`. `image:foo` and `mesh:foo`
  are deliberately distinct addresses, not aliases.

### Override after a rename

When you rename a file and want the existing address to keep
resolving, write the original address into the sidecar:

```json
// File: assets/images/icons/pointer.png (renamed from cursor.png)
{ "address": "image:icons/cursor" }
```

This is the **only** mechanism for survival across rename. No
GUIDs, no redirector files. The sidecar is the source of truth
for identity; the path is just where the bytes live.

### Asset roots

Consumers can declare named roots via
`registerAssetRoot({ id, baseUrl })`. Roots prefix the ref:
`image:@local/icons/cursor` resolves under the `@local` root's
base URL. The default root (no prefix) is the project's
`assets/` directory.

## Per-asset sidecar — `<asset>.meta.json`

Adjacent to every source asset, in VCS, text-diffable.

```jsonc
{
  // Identity. If absent, derived from path.
  "address": "image:icons/cursor",

  // Delivery profile. Defaults from vibesmith.toml's
  // [assets.defaults.<kind>]. Per-target override optional.
  "delivery": "bake",            // or "stream"
  //   or:   { "browser": "stream", "native": "bake" }

  // Tags. Power tree-shake "always include" lists, search,
  // agent queries.
  "tags": ["ui", "cursor"],

  // Kind-specific knobs. Schemas live in @vibesmith/content-schemas.
  "image": {
    "format": "auto",            // "auto" | "png" | "webp" | "avif"
    "maxWidth": 256,
    "compression": "lossy"       // "lossy" | "lossless"
  }
}
```

Validated by `@vibesmith/content-schemas` (`AssetSidecar` schema).
Missing sidecars get default-generated by `vibesmith assets import`.

## Delivery razor — `bake` vs `stream`

Two delivery profiles. Per-asset + per-target.

| Profile  | Use for                                   | Lives where                |
| -------- | ----------------------------------------- | -------------------------- |
| `bake`   | Small, hot-path, ship-once-and-forever    | Inside the install bundle  |
| `stream` | Large, optional, swappable per release    | Release channel; URL only  |

### Defaults

| Kind  | Default | Why |
|-------|---------|-----|
| image | bake    | UI icons + small textures dominate; FOUT is annoying |
| audio | stream  | Music tracks dominate the byte count; preload latency is OK |
| mesh  | stream  | Big geometry; release-channel pull is the natural shape |
| font  | bake    | FOUT is painful; fonts are small |

### Per-target overrides

`vibesmith.toml`:

```toml
[assets.defaults.image]
delivery = "bake"

[assets.targets.browser]
prefer = "stream"        # browsers favour CDN over install bytes
bake-max-bytes = 65536   # anything bigger flips bake → stream

[assets.targets.native]
prefer = "bake"          # native installs can carry more weight
bake-max-bytes = 1048576
```

### Resolution order

Most specific wins:

1. Sidecar per-target override (`"delivery": { "browser": "stream" }`)
2. Sidecar single value (`"delivery": "bake"`)
3. `[assets.defaults.<kind>]` in `vibesmith.toml`
4. `[assets.targets.<target>].prefer`

After resolution, `bake-max-bytes` flips bake → stream when the
emitted bytes exceed the cap.

## Editor — auto-population

Open a project in the editor and the Assets panel populates
immediately. You don't run a CLI subcommand; you don't restart
the editor; you don't import the asset. This is the default-
behaviour benchmark established native editors set.

How it works:

- **On project open** — the editor walks `<project>/assets/`,
  derives an address per file from the path
  (`assets/<kind-folder>/<ref>[.ext]`), reads any per-file
  `<file>.meta.json` sidecar for `address` /
  `data.schema` / `data.address-prefix` overrides, and registers
  the resulting `vibesmith/asset-registry` provider. The Assets
  panel + every `loadAsset(...)` call in the consumer's code see
  the populated registry.
- **On change** — adding / removing / renaming files (or editing
  their sidecars) under `assets/` re-walks the tree and updates
  the registry live. The Assets panel re-renders without a manual
  refresh.
- **On close** — the editor clears the registry + releases the
  blob URLs it minted.

The walker is the editor-side equivalent of
`@vibesmith/vite-plugin-assets`. Same address rules; same sidecar
overrides. The standalone Vite path remains the primary surface
for `vibesmith build`; the editor uses its in-process walker to
avoid the dependency on a running Vite dev server.

The editor walker doesn't transcode, hash content, or apply
delivery-mode optimisation — those are build-time concerns the
production pipeline handles. For dev-loop iteration, raw bytes
plus a Blob URL is enough.

## Runtime — `loadAsset(address)`

Consumer + framework code reaches for assets by address, not
path. The runtime resolves through the address registry that
the build emits.

```ts
import { loadAsset, AssetNotFoundError } from '@vibesmith/runtime';

// Returns a typed handle per kind:
const tex = await loadAsset({ kind: 'image', ref: 'icons/cursor' });
//                         ⊳ { texture: THREE.Texture, width, height, ... }

const gltf = await loadAsset({ kind: 'mesh', ref: 'characters/hero' });
//                         ⊳ { scene: THREE.Group, animations: ... }
```

### Prefetch

```ts
import { prefetchAsset } from '@vibesmith/runtime';
prefetchAsset({ kind: 'audio', ref: 'music/menu-loop' });
```

### Scoped sets (for level streaming)

```ts
import { withAssetSet } from '@vibesmith/runtime';
const cleanup = withAssetSet([
  { kind: 'mesh', ref: 'characters/hero' },
  { kind: 'audio', ref: 'sfx/footstep' },
]);
// ... later when the player leaves the area
cleanup();
```

## `AssetNotFoundError` — the self-correction loop

Every runtime asset error carries a `hint:` field naming the
literal CLI command to run. This is the contract that lets an AI
agent watching logs *correct itself*:

```ts
class AssetNotFoundError extends Error {
  address: string;
  kind: AssetKind | null;
  triedUrls: string[];
  hint: string;            // mandatory, contains the literal CLI command
  registryGeneration: number;
}
```

When an agent sees `AssetNotFoundError`:

1. Read `address` — what was the agent / consumer asking for?
2. Read `hint` — the framework tells you the next command:
   typically `vibesmith assets import <path>` or
   `vibesmith assets list --kind <kind>`.
3. Run it.
4. Reload. Success.

That's the agent self-correction loop. **Don't paper over a miss
with a try/catch; surface it.**

## CLI — `vibesmith assets …`

```sh
# Drop a source file under the right folder + write a default sidecar.
vibesmith assets import /path/to/cursor.png
#   imported /path/to/cursor.png → assets/images/cursor.png
#     address   image:cursor
#     sidecar   assets/images/cursor.png.meta.json

# Override the assigned address (e.g. after a rename).
vibesmith assets import /path/to/pointer.png --address image:icons/cursor

# Inspect.
vibesmith assets list --kind image
vibesmith assets list --delivery stream --tag music
vibesmith assets where-defined image:icons/cursor

# Rebuild the address registry (alias for the discovery + sidecar
# pass of `vibesmith build`).
vibesmith assets scan

# Confirm every loadAsset() callsite resolves against the registry.
vibesmith assets verify
```

The legacy `webgl-assets` binary stays for one minor version as a
deprecation shim — calls `vibesmith assets …` underneath, logs a
deprecation warning.

## MCP — agent surface

Five Tier-2 deferred tools (per [`mcp-tiered-surface`](./mcp-tiered-surface)).
An assistant fetches schemas on demand via `tools_find`:

```ts
vibesmith.assets.list({ kind?, tag?, delivery? })
//   → [{ address, kind, delivery, path }]

vibesmith.assets.where-defined({ address })
//   → { path, sidecarPath, references: [{ file, line }] }

vibesmith.assets.import({ sourcePath, kind?, address? })
//   → { address, sidecarPath }

vibesmith.assets.set-delivery({ address, delivery })
//   → { updatedSidecar }

vibesmith.assets.preview({ address, target? })
//   → { url, thumbnailBase64? }
```

## Importer-hook extensibility

Capability extensions register their own importers at build time:

```ts
import { registerAssetImporter } from '@vibesmith/asset-pipeline';

registerAssetImporter({
  kind: 'image',
  match: (file) =>
    file.endsWith('.png') || file.endsWith('.jpg') || file.endsWith('.webp'),
  run: async ({ source, sidecar, target }) => {
    // transform source bytes per sidecar.image.* settings
    return {
      outputs: [{ bytes, mime, suffix }],
      metadata: { width, height },
    };
  },
});
```

The framework ships built-ins for `image`, `audio`, `font`,
`mesh`. Custom importers (Spine atlas, MagicaVoxel `.vox`, etc.)
register before discovery runs.

## Why this design

A few cross-cutting choices that go against common engine
patterns. RFC 0001 § 14 lists them with full reasoning; the
short version:

- **No `.meta`-files-as-GUID-carriers (Unity).** The address
  layer carries identity; sidecars carry settings only.
- **No binary-embedded asset settings (Unreal `.uasset`).**
  Settings must be text-diffable.
- **No reference counting at runtime (Unity Addressables).** WebGC
  + per-kind dispose hook is enough.
- **No two-tier "primary vs secondary" asset model (Unreal).**
  One asset model; delivery profile carries bake-vs-stream.
- **No heavy importer class hierarchies (Unity `AssetPostprocessor`,
  Unreal `UFactory`).** Single registration function with
  `{ kind, match, run, preview }`.
- **No whole-pack-only delivery (Godot `.pck`).** Per-asset opt
  is mandatory; per-pack is opt-in for `native`-only bundles.
- **No 2D-as-secondary-citizen (Unreal Paper2D).** `image / font
  / audio` are first-class peers of `mesh`.
